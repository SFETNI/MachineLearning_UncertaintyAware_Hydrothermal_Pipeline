# Outputs Directory

## Overview

This directory stores all outputs generated by the ML pipeline, including:
- Performance metrics
- Visualizations
- Uncertainty quantification results

## Structure

### `figures/`
Generated plots and visualizations:
- Parity plots (predicted vs actual)
- Feature importance charts
- Distribution plots
- Correlation matrices
- Residual plots

### `tables/`
CSV files with quantitative results:
- Model performance metrics (R², RMSE, MAE)
- Train/test split indices
- Cross-validation scores
- Feature importance rankings
- Prediction results

### `conformal/`
Conformal prediction outputs:
- Prediction intervals
- Calibration residuals
- Coverage statistics
- Interval widths

## File Naming Convention

Files are automatically named with timestamps and descriptive labels:
- `parity_plot_{target}_{timestamp}.png`
- `performance_metrics_{mode}_{timestamp}.csv`
- `conformal_intervals_{target}_{alpha}_{timestamp}.csv`

## Storage Management

Output files are excluded from version control by default (see `.gitignore`).

To share results:
1. Commit specific result files by force-adding: `git add -f outputs/tables/metrics.csv`
2. Or include summarized results in the main README
3. Or provide links to external storage (Zenodo, Figshare, etc.)

## Reproducing Outputs

To regenerate all outputs from scratch:
1. Ensure data is in `data/` directory
2. Run the main notebook: `jupyter notebook notebooks/hydrothermal_ml_pipeline.ipynb`
3. Execute all cells
4. Outputs will be saved automatically with timestamps
